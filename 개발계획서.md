# 스마트 스케줄 노티스 (SSN) 개발 계획서

> 이 문서는 [readme.md](readme.md)의 기획 내용을 바탕으로 만든 **"앱을 어떻게 만들지" 설명서**입니다.
> 코딩을 처음 하시는 분도 이해할 수 있도록 쉬운 말로 적었습니다.

---

## 0. 한 문장 요약

> "시간표에 맞춰, 쉬는 시간마다 다음 수업 공지를 화면에 자동으로 띄우고 목소리로 읽어주는 교실용 알림판 앱"을 만듭니다.

---

## 1. 우리가 만들 앱은 어떤 거예요?

학교 교실 앞에 **똑똑한 전자 알림판**이 하나 있다고 상상해 보세요.

- 선생님이 컴퓨터로 공지("다음 수학 시간에 컴퍼스 가져오기")를 입력하면
- 그 알림판이 **쉬는 시간마다 알아서** 공지를 큰 글씨로 보여주고
- 동시에 **목소리로 읽어줘서** 학생들이 눈과 귀로 확인합니다.

선생님이 교실마다 찾아다닐 필요가 없어지고, 학생은 준비물을 깜빡하지 않게 됩니다.

---

## 2. 어떤 도구로 만들까요? (요리 재료 같은 거예요)

앱을 만드는 건 요리와 비슷합니다. 필요한 "재료(도구)"를 먼저 정리해 볼게요.

| 무엇을 만들 때 | 사용하는 도구 | 쉽게 말하면 |
| --- | --- | --- |
| 화면에 글자/버튼 보이기 | HTML | 종이에 글을 적는 것 |
| 화면을 예쁘게 꾸미기 | CSS | 글에 색칠하고 모양 잡는 것 |
| 화면이 움직이게 하기 | JavaScript | 버튼을 누르면 반응하게 만드는 마법 |
| 목소리로 읽어주기 | 브라우저 음성 기능(Web Speech API) | 컴퓨터가 글을 소리내어 읽는 기능 |
| 정보 저장하기 | 브라우저 저장공간(LocalStorage) | 메모지에 적어두고 나중에 다시 보는 것 |

> 걱정 마세요! 이 도구들은 대부분 **인터넷 브라우저(크롬 등)** 안에 이미 들어있어서 따로 설치할 게 거의 없습니다.

---

## 3. 화면은 어떻게 생겼나요?

화면을 **큰 칸 하나 + 작은 칸 둘**로 나눕니다.

```
┌─────────────────────────────────────┬───────────────┐
│                                     │   시간표 칸    │
│        큰 공지 화면 (왼쪽, 넓음)        │ (지금 몇 교시   │
│   - 공지를 큰 글씨로 보여줌            │  인지 표시)    │
│   - 5초마다 다음 공지로 바뀜          ├───────────────┤
│   - "다시 듣기" 버튼                  │  공지 목록 칸  │
│                                     │ (오늘 공지들이  │
│                                     │  쭉 나열됨)    │
└─────────────────────────────────────┴───────────────┘
         넓은 쪽 (약 3/4)                  좁은 쪽 (약 1/4)
```

- **왼쪽 큰 칸**: 지금 보여줄 공지 하나를 크게 띄웁니다.
- **오른쪽 위 칸**: 오늘의 시간표를 보여주고, 지금 진행 중인 수업을 강조합니다.
- **오른쪽 아래 칸**: 오늘과 앞으로의 공지를 목록으로 보여줍니다. 누르면 왼쪽 큰 칸에 나타나요.

## 4. 어떤 정보들을 다루나요?

앱은 크게 **두 종류의 정보**를 기억합니다. (아래는 정보를 정리하는 예시 모양입니다.)

### 4.1 시간표 정보
"1학년 3반은 월요일 1교시에 국어, 2교시에 수학" 같은 내용이에요.

```json
{
  "반": "1-3",
  "요일": "월",
  "수업": [
    { "교시": 1, "과목": "국어", "시작": "09:00", "끝": "09:45" },
    { "교시": 2, "과목": "수학", "시작": "09:55", "끝": "10:40" }
  ]
}
```

### 4.2 공지 정보
"수학 시간 준비물: 컴퍼스, 각도기" 같은 내용이에요.

```json
{
  "공지번호": "N001",
  "반": "1-3",
  "제목": "수학 수행평가 준비물",
  "내용": "컴퍼스, 각도기 지참",
  "날짜": "2026-06-08",
  "중요도": 1
}
```

> `{ }` 와 `" "` 기호가 낯설 수 있는데, 이건 **"정보를 컴퓨터가 알아보게 정리하는 표 양식"**이라고 생각하시면 됩니다. 지금 외울 필요는 없어요.

## 5. 앱이 가져야 할 기능 (해야 할 일 목록)

| 기능 | 무슨 일을 하나요? | 쉽게 풀면 |
| --- | --- | --- |
| 자동 공지 띄우기 | 쉬는 시간이 되면 다음 수업 공지를 자동으로 보여줌 | 알람시계처럼 때 되면 알려줌 |
| 화면 나누기 | 왼쪽은 공지, 오른쪽은 시간표/목록 | 칠판을 영역별로 나눠 쓰는 것 |
| 공지 자동 넘기기 | 공지를 5초 보여준 뒤 다음 공지로 바꿈 | 사진 슬라이드쇼처럼 넘김 |
| 공지 골라 보기 | 목록에서 공지를 누르면 크게 보여줌 | 메뉴판에서 음식 하나 고르는 것 |
| 목소리로 읽기 | 공지를 소리내어 읽어줌, 다시 듣기 가능 | 컴퓨터가 또박또박 읽어줌 |
| 현재 수업 표시 | 지금이 몇 교시인지 강조해서 보여줌 | "지금 여기!" 형광펜 칠하기 |

## 6. 언제 무엇을 만들까요? (8주 계획)

처음부터 다 만들지 않고, **한 주에 한 단계씩** 차근차근 만듭니다.

| 주차 | 이번 주에 할 일 | 다 하면 이렇게 됩니다 |
| --- | --- | --- |
| 1주차 | 앱 그림 그리고 계획 세우기 | 어떤 화면이 필요한지 종이에 그림 완성 |
| 2주차 | 화면 틀(레이아웃) 만들기 | 왼쪽 큰 칸 + 오른쪽 작은 칸 모양 완성 |
| 3주차 | 정보 정리 방법 만들기 | 공지를 적으면 목록에 추가됨 |
| 4주차 | 핵심 기능 ① 만들기 | 5초마다 공지 넘어가고, 눌러서 골라보기 됨 |
| 5주차 | 핵심 기능 ② 만들기 | 목소리로 읽어주고, 때 되면 자동 알림 뜸 |
| 6주차 | 기능들 합치기 | 모든 기능이 하나로 잘 동작하는지 확인 |
| 7주차 | 잘못된 부분 고치기 | 끊기거나 겹치는 문제 해결, 화면 다듬기 |
| 8주차 | 마무리 및 기록하기 | 완성! 만들면서 겪은 일 정리해 문서로 남김 |

---

## 7. 중간 점검 지점 (여기까지 오면 절반 성공!)

- **2주차**: 화면 모양이 완성되면 → "앱처럼 보이기 시작!"
- **5주차**: 자동 넘김 + 목소리 + 자동 알림이 되면 → "핵심 기능 완성!"
- **6주차**: 모든 기능이 합쳐지면 → "거의 다 됐다!"
- **8주차**: 오류까지 고치면 → "완성! 발표 준비 끝!"

## 8. 미리 알아두면 좋은 어려움들 (그리고 해결법)

| 생길 수 있는 문제 | 왜 문제예요? | 어떻게 해결하나요? |
| --- | --- | --- |
| 공지가 여러 개 겹칠 때 | 어떤 걸 먼저 보여줄지 헷갈림 | "중요한 것 먼저" 규칙을 정해둠 |
| 목소리가 자꾸 끊김 | 읽다가 다른 공지가 끼어듦 | 새로 읽기 전에 앞 소리를 멈춤 |
| 시간이 안 맞아 알림이 안 뜸 | 컴퓨터 시계와 시간표가 어긋남 | 1분마다 시간을 다시 확인하게 함 |
| 새로고침하면 공지가 사라짐 | 정보를 저장 안 해둠 | 브라우저에 저장해두고 다시 불러옴 |

> 이런 문제는 **처음 만드는 사람이라면 누구나 겪는 일**이에요. 미리 알고 있으면 당황하지 않습니다.

---

## 9. 나중에 더 해보면 좋은 것들 (지금은 안 해도 돼요)

- 선생님이 여러 반에 공지를 **한 번에 보내는** 기능
- 정보를 인터넷 서버에 저장해 **어디서든** 보는 기능
- 학생이 공지를 **읽었는지 확인**하는 기능
- 공지에 **사진**을 같이 올리는 기능
