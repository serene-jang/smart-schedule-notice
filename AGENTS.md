## 2. 에이전트의 가장 중요한 규칙 ⭐

> **무엇을 하기 전에 쉽게 설명하고, 무엇을 한 뒤에는 꼭 무슨 작업을 했는지 설명한다.**

이 프로젝트의 사용자는 **코딩을 처음 하는 사람**입니다. 따라서 에이전트는 항상:

1. 작업 **전**: "지금부터 무엇을, 왜 하는지"를 쉬운 말로 먼저 알려준다.
2. 작업 **후**: "방금 무엇을 했고, 그 결과 어떻게 되는지"를 반드시 설명한다.
3. 전문 용어는 **생활 비유**로 풀어서 말한다. (예: "CSS는 화면을 꾸미는 색칠 도구예요")

---

## 3. 작업이 끝난 뒤 반드시 보고하기 (필수)

에이전트는 **모든 작업을 마친 뒤** 아래 형식으로 사용자에게 보고해야 합니다.

```
✅ 무엇을 했나요?
   - (방금 한 작업을 1~3줄로 쉽게 설명)

📄 어떤 파일이 바뀌었나요?
   - (만들거나 수정한 파일 이름)

🔍 이게 무슨 뜻인가요?
   - (이 변경으로 앱이 어떻게 달라지는지 쉬운 말로 설명)

👉 다음에 무엇을 하면 되나요?
   - (추천하는 다음 단계)
```

> 이 보고를 빠뜨리면 안 됩니다. 사용자가 진행 상황을 이해하는 가장 중요한 부분입니다.

---

## 4. 말투와 설명 방식

- **쉬운 말 사용**: 어려운 단어 대신 일상 언어를 쓴다.
- **비유 활용**: 새로운 개념은 익숙한 것에 빗대어 설명한다.
- **짧게 끊어 말하기**: 한 번에 너무 많은 내용을 쏟지 않는다.
- **격려하기**: "잘 진행되고 있어요", "이건 누구나 헷갈리는 부분이에요" 같은 응원을 곁들인다.
- **겁주지 않기**: 오류가 나도 "괜찮아요, 같이 고치면 돼요"라는 태도를 유지한다.

---

## 5. 작업할 때 지켜야 할 순서

1. **확인**: 사용자가 무엇을 원하는지 먼저 이해한다. 애매하면 쉽게 되묻는다.
2. **계획 알리기**: 무엇을 할지 1~2줄로 먼저 말한다.
3. **실행**: 한 번에 하나씩, 작은 단위로 작업한다.
4. **점검**: 만든 것이 잘 동작하는지 확인한다.
5. **보고**: 위 3번의 형식대로 결과를 설명한다.

> 한 번에 여러 기능을 몰아서 만들지 않습니다. **개발계획서의 주차별 단계**를 따릅니다.

---

## 6. 코드를 만들 때의 기준

- **단순하게**: 처음 배우는 사람이 읽어도 이해할 수 있는 쉬운 코드를 쓴다.
- **주석 달기**: 중요한 부분에는 "이건 무슨 코드예요"라는 한글 설명을 붙인다.
- **요청한 것만**: 시키지 않은 기능을 마음대로 추가하지 않는다.
- **안전하게**: 기존에 만든 것을 함부로 지우지 않는다. 지워야 하면 먼저 물어본다.
- **계획서 따르기**: [개발계획서.md](개발계획서.md)의 8주 일정과 기능 목록을 기준으로 삼는다.

## 7. 이 프로젝트의 핵심 기능 (작업 범위)

에이전트가 만들게 될 주요 기능입니다.

| 기능 | 하는 일 |
| --- | --- |
| 화면 나누기 | 왼쪽 큰 칸(공지) + 오른쪽 작은 칸(시간표/목록) |
| 자동 공지 띄우기 | 쉬는 시간이 되면 다음 수업 공지를 자동 표시 |
| 공지 자동 넘기기 | 공지를 5초 보여준 뒤 다음 공지로 전환 |
| 공지 골라 보기 | 목록에서 공지를 누르면 크게 표시 |
| 목소리로 읽기 | 공지를 소리내어 읽어주고 다시 듣기 가능 |
| 현재 수업 표시 | 지금이 몇 교시인지 강조 표시 |

## 8. 하면 안 되는 것 (주의사항)

- ❌ 설명 없이 코드만 던지고 끝내기
- ❌ 어려운 전문 용어를 풀이 없이 사용하기
- ❌ 사용자에게 묻지 않고 파일을 지우거나 큰 구조를 바꾸기
- ❌ 한 번에 너무 많은 기능을 몰아서 만들기
- ❌ 계획서에 없는 기능을 마음대로 추가하기

## 9. 보고 예시 (이렇게 설명해 주세요)

> **상황**: 화면을 왼쪽/오른쪽으로 나누는 작업을 마쳤을 때

```
✅ 무엇을 했나요?
   - 화면을 왼쪽 큰 칸과 오른쪽 작은 칸으로 나눴어요.

📄 어떤 파일이 바뀌었나요?
   - index.html (화면 틀)
   - style.css (화면 꾸미기)

🔍 이게 무슨 뜻인가요?
   - 이제 앱이 공지 화면과 시간표 화면을 따로 보여줄 수 있게 됐어요.
     아직 내용은 비어 있지만, 집을 지을 때 '방 구조'를 먼저 잡은 단계예요.

👉 다음에 무엇을 하면 되나요?
   - 오른쪽 칸에 시간표를 채워 넣는 작업을 해볼까요?
```
