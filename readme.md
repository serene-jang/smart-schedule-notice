# 프로젝트명
스마트 스케줄 노티스 (Smart Schedule Notice, SSN)

## 앱 설명
**스마트 스케줄 노티스(SSN)**는 학급의 시간표와 연동하여 쉬는 시간마다 자동으로 공지사항을 화면에 띄워주고 음성으로 읽어주는 교실용 스마트 알림 시스템입니다. 

선생님은 교무실에서 한 번 클릭으로 여러 학급에 공지를 전달할 수 있고, 학생들은 시각과 청각을 동시에 활용하여 공지 내용을 놓치지 않을 수 있습니다. 특히 준비물 안내, 시험 공지, 행사 알림 등을 정확하고 신속하게 전달할 수 있어, 학급 운영 효율성을 획기적으로 높일 수 있습니다.

이 앱은 **HTML, CSS, JavaScript**만을 사용한 가볍고 빠른 웹 앱으로, 추가 설치 없이 모든 컴퓨터와 스마트보드에서 바로 실행할 수 있습니다.

## 앱의 주요 장점
✨ **선생님 입장**
- **시간 절약**: 여러 학급을 일일이 방문하지 않고 한 곳에서 모든 반에 공지 전달
- **정확한 전달**: 공지 내용이 왜곡되지 않고 정확하게 학생에게 전달됨
- **자동화된 일정 관리**: 학급별 시간표와 연동되어 공지 타이밍이 자동 설정됨
- **학생 참여 증대**: 음성과 시각 정보로 학생의 주의력 집중

✨ **학생 입장**
- **놓치지 않는 공지**: 쉬는 시간마다 화면과 음성으로 중요한 정보 전달받음
- **준비물 까먹지 않기**: 다음 수업 준비물을 미리 알 수 있어 수업 준비가 철저함
- **자기주도적 학습**: 공지 내용을 직접 확인하여 책임감 있는 학교생활
- **다국어 지원 가능**: 나중에 음성 언어를 추가하여 국제학생도 이용 가능

✨ **학교 운영 입장**
- **저비용 고효율**: 추가 기계나 소프트웨어 구입 없이 웹으로만 운영 가능
- **유지보수 간편**: 클라우드 기반으로 어디서나 업데이트 가능
- **통합 공지 시스템**: 모든 학급이 같은 플랫폼을 사용하여 일관성 유지

## 문제 정의
현재 중학교에서는 교과 선생님이 여러 학급에 같은 공지사항을 전달하기 위해 매번 쉬는 시간마다 학생을 부르거나 직접 교실을 방문해야 하는 번거로움이 있습니다. 또한, 학생들은 전달받은 공지를 잊어버리거나 다른 학급 친구에게 전달하는 과정에서 정보가 바뀌게 될 위험이 있어, 학급별 시간표와 연결된 자동 공지 전달 시스템이 필요합니다.

## 핵심 기능
- **시간표 연동 자동 팝업**: 학급별 시간표와 연결하여, 쉬는 시간 시작 시점에 맞춰 다음 교시 준비물이나 공지사항을 화면에 자동으로 띄워줌.
- **지능형 화면 분할 UI**: 화면의 1/4은 시간표와 전체 공지 리스트(스크롤 가능)를 항상 보여주고, 3/4은 선택된 공지를 5초간 집중적으로 보여주는 효율적인 정보 화면.
- **TTS(Text-to-Speech) 자동/수동 안내**: 공지 안내 시 자동으로 내용을 읽어주어 시각과 청각을 동시에 집중하게 하고, 필요할 때 언제든 다시 들을 수 있는 수동 재생 기능.

## 기대 효과
- **교사 기대효과**: 여러 반을 일일이 찾아다니는 시간 낭비를 줄이고, 한 번의 입력으로 원하는 학급에 정확히 공지를 전달하여 업무 효율성이 높아짐.
- **학생 기대효과**: 쉬는 시간마다 다음 수업 정보를 눈과 귀로 확인하여 수업 준비물을 깜박하는 것을 방지하고 자기주도적 학습 환경을 조성.

## 핵심 화면
- **메인 공지 보드 (왼쪽 3/4)**: 선택된 공지사항이 큰 글씨와 이미지로 나타나는 공간. 자동 전환 모드에서는 5초간 강조된 후 다음 우선순위 공지로 변경됨.
- **시간표와 공지사항 리스트 공간 (오른쪽 1/4)**: 각 반 시간표와 오늘 및 오늘 이후의 공지사항이 리스트로 나타남.
  - 상단: 현재 학급의 일일 시간표 (현재 교시 강조).
  - 하단: 오늘 및 다음의 공지가 등록된 모든 공지사항 리스트.

## 페르소나
- **이름**: 김지훈 (30대, 중학교 수학 교사)
- **상황**: 6개 학급의 수업을 담당하며 매주 수행평가 준비물을 공지해야 함. 매번 학생을 부르는 것이 미안하고 번거로움.
- **원하는 것**: "교무실에서 한 번 클릭으로 모든 담당 학급 게시판에 준비물을 띄우고 싶어요."

## �📚 개발 배경

### 왜 이 앱을 만들었나요?

중학교 현장에서는 매일 다음과 같은 문제가 반복되고 있습니다:

1. **교사의 번거로움**
   - 6개 학급을 담당하는 교사는 쉬는 시간마다 공지를 하기 위해 여러 교실을 다녀야 함
   - 매번 학생을 부르거나 들어가서 직접 말해야 해서 비효율적
   - 같은 내용을 반복 설명하느라 업무 시간 낭비

2. **학생의 정보 손실**
   - 전달받은 공지를 까먹고 다음 수업을 못 준비함
   - 친구에게 듣는 과정에서 정보가 왜곡됨
   - 시각적, 청각적 자극이 부족해서 주의가 산만함

3. **학교 운영의 비효율성**
   - 공지 시스템이 체계적이지 않아 정보 전달에 오류 발생
   - 대면 공지는 시간과 공간의 제약이 큼

**스마트 스케줄 노티스**는 이런 문제들을 **자동화된 시스템**으로 해결하기 위해 만들어졌습니다.

## 🔄 기존 방식 vs 새로운 방식

### 📋 비교 표

| 항목 | 기존 방식 (SSN 미사용) | 새로운 방식 (SSN 사용) |
|------|------|------|
| **공지 전달 방법** | 선생님이 직접 학급 방문 후 구두로 설명 | 시스템이 자동으로 화면과 음성으로 표시 |
| **소요 시간** | 6개 학급 × 5분 = 30분 | 한 번 입력으로 완료 (1분 이내) |
| **정보 정확성** | 각 학급마다 전달이 다를 수 있음 | 모든 학급이 동일한 내용을 같은 방식으로 전달받음 |
| **학생의 주의도** | 낮음 (배경음, 딴짓 가능) | 높음 (시각 + 음성 동시 자극) |
| **공지 기억력** | 낮음 (대부분 까먹음) | 높음 (반복 표시와 음성으로 강화) |
| **준비물 준비율** | 60~70% (많이 빠뜨림) | 90% 이상 (다음 수업 정보 미리 확인) |
| **학급 운영 효율** | 낮음 (공지 시간이 길어짐) | 높음 (수업 시간에 영향 없음) |
| **학교 전산 비용** | 별도 장비 구입 필요 | 웹 기반이라 추가 비용 없음 |

### 📊 시간 비교 예시

**선생님 김지훈이 6개 학급에 수행평가 준비물을 공지하는 경우:**

```
⏰ 기존 방식:
  - 1반 방문 및 설명: 5분
  - 2반 방문 및 설명: 5분
  - 3반 방문 및 설명: 5분
  - 4반 방문 및 설명: 5분
  - 5반 방문 및 설명: 5분
  - 6반 방문 및 설명: 5분
  ────────────────────────
  ⏱️ 총 30분 소요

✨ 새로운 방식 (SSN):
  - 앱에 공지 입력: 2분
  - 6개 학급에 자동 전송 완료: 즉시
  ────────────────────────
  ⏱️ 총 2분 소요 (28분 절약!)
```

## 🎬 앱 동작 과정

### 1️⃣ 시작 단계

```
📱 앱 실행
   ↓
🕐 현재 시간과 학급 시간표 로드
   ↓
📌 오늘의 공지사항 목록 표시
```

### 2️⃣ 일반적인 흐름 (예: 오전 10:20 분)

```
🎓 학생 화면 켜짐
   ├─ 오른쪽: 현재 시간표 표시
   │  └─ "1교시 (9:00~10:00) 끝남" 강조
   │
   └─ 왼쪽: "다음 2교시 준비물" 크게 표시
      └─ 공지 내용 + 음성 자동 재생

⏰ 5초 후
   ↓
→ 다음 공지로 자동 전환
```

### 3️⃣ 공지 상세 과정

```
선생님 화면
   ↓
📝 공지 입력 (제목, 내용, 대상 학급, 전송 시간)
   ↓
✅ "전송" 버튼 클릭
   ↓
🔔 해당 시간에 학생 화면에 자동 팝업
   ├─ 텍스트로 표시
   ├─ 음성으로 읽어줌 (자동)
   └─ 수동 재생 버튼 (다시 듣기 가능)
   ↓
👆 학생이 리스트에서 공지 클릭
   ↓
🔊 다시 음성 재생 가능
```

### 4️⃣ 공지 리스트 관리

```
오른쪽 사이드바 - 공지 리스트
   ├─ 📌 우선순위 높은 공지 (최상단)
   ├─ 📌 시간순 공지
   └─ 📌 스크롤로 과거 공지 확인 가능
```
